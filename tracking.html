Object Tracking

Track objects through video by preserving identity across frames. Tracking connects object detection with temporal reasoning, motion prediction, and data association.

What is Object Tracking?

Object tracking estimates the location of one or more targets over time. A tracker receives a video stream and maintains object identities even as targets move, overlap, disappear briefly, or change appearance.

Definition: A tracking output is a sequence of object states, usually bounding boxes or masks with persistent track IDs. The key challenge is not only where the object is, but whether it is still the same object across frames.

Single-object tracking

Starts with one target selected in the first frame, then follows that target through the video. Classical examples include correlation filters and Siamese-network trackers.

Multi-object tracking

Tracks many objects at once, usually by detecting objects in each frame and associating detections with existing tracks using motion and appearance cues.

Tracking Pipeline

A practical multi-object tracker is often a loop of detection, prediction, association, update, and track management.

1. Detect

Run an object detector such as YOLO, Faster R-CNN, or DETR to produce candidate boxes for each frame.

2. Predict

Use a motion model such as a Kalman filter to estimate where each active track should appear next.

3. Associate

Match new detections to existing tracks using IoU, appearance embeddings, or a combined cost matrix.

4. Update

Refresh matched tracks, create new tracks for unmatched detections, and retire stale tracks after missed frames.

Tracker Families

Family Core Idea Examples Best Use
Correlation filters Learn a fast template-like filter around the selected object. KCF, CSRT, MOSSE Fast single-object tracking
Tracking-by-detection Detect objects per frame, then associate detections over time. SORT, DeepSORT, ByteTrack Multi-object tracking in surveillance, traffic, sports
Siamese trackers Compare the target template with search regions in later frames. SiamFC, SiamRPN, SiamMask Robust single-object tracking with deep features
Joint detection and tracking Predict detections and track identities inside one learned model. CenterTrack, TrackFormer End-to-end research systems

Evaluation Metrics

Tracking metrics measure localization quality and identity consistency over time.

MOTA

Multi-Object Tracking Accuracy combines false positives, missed detections, and identity switches into one score.

IDF1

Measures how well predicted identities match ground-truth identities across the full video sequence.

HOTA

Balances detection quality and association quality, making identity errors easier to diagnose.

Practical warning: A tracker can have good boxes but poor identities. Always inspect identity switches, fragmented tracks, and long occlusions instead of relying on one aggregate score.

How Tracking Connects to CSE445

Tracking extends object detection from images to video. Students should first understand bounding boxes and mAP, then move to temporal association, Kalman filtering, and identity metrics.
