Self-Supervised Segmentation

Self-supervised segmentation uses unlabeled images, learned representations, and pseudo masks to reduce the dependence on expensive pixel-level annotation.

Why Annotation Cost Matters

Dense segmentation labels are expensive because annotators must trace boundaries or assign labels at pixel level. In medical imaging, agriculture, remote sensing, and industrial inspection, expert review may be required. Self-supervised learning helps by using unlabeled images before the final supervised segmentation stage.

Definition: Self-supervised learning creates supervision from the data itself, such as predicting missing image patches, matching augmented views, or distilling representations from a teacher network.

Core Ideas for Self-Supervised Segmentation

Contrastive Learning

Learns invariance by pulling related augmented views together and pushing unrelated examples apart. For segmentation, local spatial detail must be preserved.

Masked Image Modeling

Hides image patches and trains the model to reconstruct missing content, encouraging contextual and structural features useful for dense prediction.

DINO-style Learning

Uses teacher-student self-distillation. Vision transformer attention can reveal object-like regions that are helpful for pseudo-mask discovery.

SAM-assisted Pseudo-labeling

Uses promptable or automatic masks as annotation candidates, then filters and maps them to task-specific semantic classes.

Positive pair xia, xib = two augmented views of the same image or region Dense prediction goal Learn features that are invariant enough for recognition and spatial enough for boundaries
Intuition: A good segmentation encoder should understand that two augmented views contain the same structure while still keeping enough local detail to recover boundaries.

Suggested Pipeline

Pretrain Encoder

Use unlabeled domain images with contrastive learning, masked image modeling, or teacher-student representation learning.

Generate Pseudo Masks

Use clustering, attention maps, SAM prompts, or a weak teacher model to propose candidate masks.

Fine-tune Segmentation Head

Train a decoder or mask head with available labels and carefully filtered pseudo labels.

Evaluate Quality

Measure IoU, Dice, AP, or PQ on a trusted validation set with human-reviewed labels.

Pitfall: Object-like attention maps and promptable masks are not automatically verified labels. Pseudo masks should be filtered, inspected, and validated before being treated as training targets.

SimCLR-style Encoder Pretraining

Libraries: PyTorch, Torchvision. This sketch trains an encoder to make two augmented views of the same image close in embedding space.

import torch
import torch.nn.functional as F
from torchvision.models import resnet18

encoder = resnet18(weights=None)
encoder.fc = torch.nn.Identity()  # use features instead of classifier logits
projection = torch.nn.Sequential(
    torch.nn.Linear(512, 256),
    torch.nn.ReLU(),
    torch.nn.Linear(256, 128)
)

def nt_xent_loss(z1, z2, temperature=0.2):
    z1 = F.normalize(z1, dim=1)
    z2 = F.normalize(z2, dim=1)
    logits = z1 @ z2.T / temperature
    labels = torch.arange(z1.size(0), device=z1.device)
    return F.cross_entropy(logits, labels)

for view_a, view_b in unlabeled_loader:
    view_a, view_b = view_a.to(device), view_b.to(device)

    z_a = projection(encoder(view_a))
    z_b = projection(encoder(view_b))
    loss = nt_xent_loss(z_a, z_b)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

Explanation: After pretraining, remove the projection head and transfer the encoder into a segmentation model. This works best when the unlabeled images come from the target domain.

DINO Feature Extraction for Dense Prediction

Libraries: Transformers, PyTorch, Pillow. DINO features can be used for clustering, visualization, or initializing dense prediction models.

from PIL import Image
import torch
from transformers import AutoImageProcessor, AutoModel

image = Image.open("unlabeled_image.jpg").convert("RGB")

processor = AutoImageProcessor.from_pretrained("facebook/dinov2-base")
model = AutoModel.from_pretrained("facebook/dinov2-base")

inputs = processor(images=image, return_tensors="pt")

with torch.no_grad():
    outputs = model(**inputs)

# Patch tokens carry spatial information from the image.
patch_tokens = outputs.last_hidden_state[:, 1:, :]
features = torch.nn.functional.normalize(patch_tokens, dim=-1)
print(features.shape)

Explanation: Patch-level features can reveal object structure. For segmentation, reshape patch tokens into a feature grid and train a decoder or cluster features into candidate regions.

Masked Autoencoder Pretraining Skeleton

Libraries: PyTorch. This simplified skeleton shows the learning objective: reconstruct masked visual content from visible patches.

import torch
import torch.nn.functional as F

def random_patch_mask(patches, mask_ratio=0.75):
    batch, num_patches, dim = patches.shape
    keep_count = int(num_patches * (1 - mask_ratio))
    noise = torch.rand(batch, num_patches, device=patches.device)
    keep_ids = noise.argsort(dim=1)[:, :keep_count]
    return keep_ids

for images in unlabeled_loader:
    images = images.to(device)
    patches = patch_embed(images)
    keep_ids = random_patch_mask(patches)

    visible = torch.gather(
        patches,
        dim=1,
        index=keep_ids.unsqueeze(-1).expand(-1, -1, patches.size(-1))
    )

    latent = encoder(visible)
    reconstructed = decoder(latent, keep_ids)
    loss = F.mse_loss(reconstructed, patches)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

Explanation: Masked image modeling encourages the encoder to learn context and structure. The pretrained encoder can then be attached to a segmentation decoder.

Pseudo-mask Fine-tuning Pipeline

Libraries: PyTorch. This example mixes trusted human labels with filtered pseudo masks during semantic segmentation training.

import torch
import torch.nn.functional as F

model.train()

for batch in train_loader:
    images = batch["image"].to(device)
    labels = batch["mask"].to(device)
    pseudo = batch["pseudo_mask"].to(device)
    pseudo_conf = batch["pseudo_confidence"].to(device)

    logits = model(images)

    supervised_loss = F.cross_entropy(logits, labels, ignore_index=255)

    # Only trust pseudo labels above a confidence threshold.
    valid_pseudo = pseudo_conf > 0.85
    pseudo_loss_map = F.cross_entropy(logits, pseudo, reduction="none")
    pseudo_loss = pseudo_loss_map[valid_pseudo].mean()

    loss = supervised_loss + 0.4 * pseudo_loss

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

Explanation: Pseudo labels can improve data efficiency, but low-quality masks can damage training. Confidence filtering and a trusted validation set are essential.
