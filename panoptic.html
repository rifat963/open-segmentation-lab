Panoptic Segmentation

Panoptic segmentation unifies semantic and instance segmentation. Every pixel receives a semantic class, and countable objects also receive instance identities.

Panoptic Segmentation Definition

A panoptic output assigns every pixel to exactly one segment. Background-like regions such as road, sky, grass, tissue, or soil are labeled as stuff. Countable objects such as people, cars, fruits, cells, or tools are labeled as things with instance identities.

Pixel assignment P(i,j) = (c, id), id = 0 for stuff, id > 0 for thing instances Constraint ∀ (i,j): exactly one segment covers pixel (i, j)
Definition: Panoptic segmentation is a unified scene parsing task: it labels stuff regions and separates thing instances in one non-overlapping output.

Things vs Stuff Classes

The thing-stuff distinction is central to panoptic datasets. Things are countable objects with individual identities. Stuff is amorphous material or background context that usually does not need an instance id.

Things

Cars, pedestrians, leaves, fruits, tools, animals, lesions, and cell nuclei. These categories can be counted and separated into individual instances.

Stuff

Road, sky, water, soil, grass, wall, tissue background, and other continuous regions. These categories are labeled as semantic regions.

Intuition: If asking "how many?" makes sense, the category is often a thing. If the region is better described as material, surface, or background, it is often stuff.

Relationship with Semantic and Instance Segmentation

Panoptic segmentation can be understood as a disciplined merge of semantic segmentation and instance segmentation. It keeps the full pixel coverage of semantic segmentation while preserving individual identities for thing classes.

Task Stuff classes Thing instances Complete pixel map
SemanticYesNoYes
InstanceNo, usually foreground objects onlyYesNo
PanopticYesYesYes

Panoptic Quality

Panoptic Quality evaluates recognition and mask quality together. It matches predicted and ground-truth segments, measures overlap for matched pairs, and penalizes missed or extra segments.

Panoptic Quality PQ = SQ × RQ Segmentation Quality SQ = (1/|TP|) Σ(p,g) ∈ TP IoU(p, g) Recognition Quality RQ = TP / (TP + ½ FP + ½ FN)
Exam note: PQ is not just IoU. It combines the quality of predicted masks with the model's ability to find the correct set of segments.

Applications

Autonomous Driving

Unify lane, road, sky, person, vehicle, sign, and sidewalk understanding in one scene representation.

Robotics

Support navigation and manipulation by separating objects while retaining layout and background context.

Agriculture

Distinguish crop rows, soil, weeds, fruits, leaves, and individual plants for field analytics.

Medical Imaging

Represent tissue regions and countable anatomical or cellular instances in a single prediction map.

PyTorch Tutorial Lab

This step-by-step lab covers the complete panoptic segmentation pipeline — building non-overlapping panoptic targets, loading a universal segmentation model, running the training loop with combined losses, and computing Panoptic Quality on a validation set.

Step 1 — Panoptic Label Format

A panoptic label assigns every pixel exactly one segment. Each segment has a semantic class id, and thing segments also have a unique instance id. Stuff segments share one region per class with instance id 0. This step shows how to build and validate panoptic target tensors from raw annotation files.

ⓘ No overlaps allowed: Every pixel must belong to exactly one segment. Verify this with a uniqueness check before training — even a single overlapping pixel in your labels will silently corrupt the matching loss. 
import torch
import numpy as np
from PIL import Image
from pathlib import Path


# ---------------------------------------------------------------
# Panoptic target structure
# ---------------------------------------------------------------
# semantic_map : (H, W) int64  — class id at every pixel
# instance_map : (H, W) int64  — instance id at every pixel
#                                0 for stuff, >0 for each thing instance
# segments_info: list of dicts — one entry per unique segment
# ---------------------------------------------------------------

HEIGHT, WIDTH = 512, 512

# --- Example: manually building a panoptic target ---

# Start with all pixels assigned to the background stuff class (id=0, instance=0).
semantic_map = torch.zeros((HEIGHT, WIDTH), dtype=torch.long)
instance_map = torch.zeros((HEIGHT, WIDTH), dtype=torch.long)

# Stuff region: road covering the bottom third of the image.
# Stuff uses instance_id=0 — there is only one road region.
road_mask                 = torch.zeros((HEIGHT, WIDTH), dtype=torch.bool)
road_mask[340:, :]        = True
semantic_map[road_mask]   = 7    # class 7 = road
instance_map[road_mask]   = 0    # stuff has no instance id

# Stuff region: sky covering the top quarter of the image.
sky_mask                  = torch.zeros((HEIGHT, WIDTH), dtype=torch.bool)
sky_mask[:130, :]         = True
semantic_map[sky_mask]    = 10   # class 10 = sky
instance_map[sky_mask]    = 0

# Thing instance 1: first car
car_1_mask                = torch.zeros((HEIGHT, WIDTH), dtype=torch.bool)
car_1_mask[220:300, 60:200] = True
semantic_map[car_1_mask]  = 2    # class 2 = car
instance_map[car_1_mask]  = 1    # first car gets instance_id=1

# Thing instance 2: second car (different location, same class)
car_2_mask                = torch.zeros((HEIGHT, WIDTH), dtype=torch.bool)
car_2_mask[230:310, 300:460] = True
semantic_map[car_2_mask]  = 2    # same class id as first car
instance_map[car_2_mask]  = 2    # but different instance id

# Thing instance 3: a pedestrian
ped_mask                  = torch.zeros((HEIGHT, WIDTH), dtype=torch.bool)
ped_mask[200:350, 470:520] = True
semantic_map[ped_mask]    = 1    # class 1 = person
instance_map[ped_mask]    = 3

# --- Build segments_info metadata ---
# This mirrors the COCO panoptic annotation format.
segments_info = [
    {"id": 0, "category_id": 7,  "is_thing": False, "area": int(road_mask.sum())},
    {"id": 0, "category_id": 10, "is_thing": False, "area": int(sky_mask.sum())},
    {"id": 1, "category_id": 2,  "is_thing": True,  "area": int(car_1_mask.sum())},
    {"id": 2, "category_id": 2,  "is_thing": True,  "area": int(car_2_mask.sum())},
    {"id": 3, "category_id": 1,  "is_thing": True,  "area": int(ped_mask.sum())},
]

target = {
    "semantic_map":  semantic_map,   # (H, W) class ids
    "instance_map":  instance_map,   # (H, W) instance ids
    "segments_info": segments_info,
}

# --- Validation: every pixel must be covered exactly once ---
# Verify no pixel was left unassigned (still at the initial class 0 unless road/sky covers it)
total_assigned = (road_mask | sky_mask | car_1_mask | car_2_mask | ped_mask).sum()
print(f"Pixels assigned: {total_assigned.item()} / {HEIGHT * WIDTH}")

# Verify thing masks do not overlap each other
thing_overlap = (car_1_mask & car_2_mask).any() or \
                (car_1_mask & ped_mask).any()   or \
                (car_2_mask & ped_mask).any()
print(f"Thing mask overlap detected: {thing_overlap}")   # should be False

# Unique (class_id, instance_id) pairs in the label
unique_segments = set()
for cls, inst in zip(semantic_map.flatten().tolist(),
                     instance_map.flatten().tolist()):
    unique_segments.add((cls, inst))
print(f"Unique segments in target: {len(unique_segments)}")
  • Every pixel must be assigned to exactly one segment — overlapping masks will cause the matching loss to assign incorrect targets during training.
  • Stuff regions use instance_id=0; thing instances use unique positive integers starting from 1.
  • Two thing instances of the same class (e.g., car 1 and car 2) share the same category_id but have different instance_id values.
  • Always run a validation check on a sample of your annotations before training — panoptic label errors are silent but severely degrade PQ.

Step 2 — Model Architecture & Panoptic Inference

Mask2Former is the current standard for universal segmentation including panoptic tasks. It uses a masked cross-attention transformer decoder with a Swin Transformer backbone to predict a set of mask queries, then post-processing assembles the final panoptic output.

ⓘ Post-processing is mandatory: Mask2Former's raw output is a set of query masks and class logits — not a panoptic map. Always use post_process_panoptic_segmentation from the image processor to convert them into a usable segment map. 
from PIL import Image
import torch
from transformers import (
    Mask2FormerImageProcessor,
    Mask2FormerForUniversalSegmentation,
)


# ---------------------------------------------------------------
# Load model and processor
# ---------------------------------------------------------------
# Available panoptic checkpoints (COCO-trained):
#   facebook/mask2former-swin-tiny-coco-panoptic   (~47M params, ~4 GB GPU)
#   facebook/mask2former-swin-small-coco-panoptic  (~69M params, ~6 GB GPU)
#   facebook/mask2former-swin-large-coco-panoptic  (~216M params, ~14 GB GPU)
MODEL_NAME = "facebook/mask2former-swin-small-coco-panoptic"

processor = Mask2FormerImageProcessor.from_pretrained(MODEL_NAME)
model     = Mask2FormerForUniversalSegmentation.from_pretrained(MODEL_NAME)
model.eval()

# ---------------------------------------------------------------
# Single-image inference
# ---------------------------------------------------------------
image  = Image.open("street_scene.jpg").convert("RGB")

# Processor resizes, normalizes, and pads the image for the model.
inputs = processor(images=image, return_tensors="pt")

with torch.no_grad():
    outputs = model(**inputs)

# ---------------------------------------------------------------
# Post-process into a panoptic segmentation map
# ---------------------------------------------------------------
# target_sizes: original (height, width) BEFORE processor resize.
# The processor will upsample predictions back to this resolution.
panoptic_result = processor.post_process_panoptic_segmentation(
    outputs,
    target_sizes=[image.size[::-1]],   # PIL image.size = (W, H), flip to (H, W)
)[0]

# segmentation: (H, W) integer tensor — each unique value is a segment id
seg_map       = panoptic_result["segmentation"]

# segments_info: list of dicts, one per unique segment id in seg_map
# Each dict contains: id, label_id, was_fused, score
segments_info = panoptic_result["segments_info"]

print(f"Segmentation map shape: {seg_map.shape}")
print(f"Number of segments predicted: {len(segments_info)}")

# Inspect first 3 segments
for seg in segments_info[:3]:
    print(f"  Segment id={seg['id']}  class={seg['label_id']}  "
          f"is_thing={model.config.id2label[seg['label_id']]}")

# ---------------------------------------------------------------
# Separate thing instances from stuff regions
# ---------------------------------------------------------------
thing_ids = set(model.config.id2label[seg["label_id"]]
                for seg in segments_info
                if seg["label_id"] in model.config.thing_class_ids)

stuff_segments = [s for s in segments_info
                  if s["label_id"] not in model.config.thing_class_ids]
thing_segments = [s for s in segments_info
                  if s["label_id"] in model.config.thing_class_ids]

print(f"\nStuff regions: {len(stuff_segments)}")
print(f"Thing instances: {len(thing_segments)}")
  • Pass target_sizes as the original image resolution before any processor resizing — the processor upsamples predictions to match.
  • The segmentation map uses segment ids, not class ids. Look up class ids via segments_info.
  • Use model.config.thing_class_ids to separate thing instances from stuff regions in the post-processed output.
  • Set model.eval() and use torch.no_grad() during inference to disable dropout and gradient computation.

Step 3 — Training Loop

Universal segmentation models like Mask2Former use three combined losses: cross-entropy for class prediction, binary cross-entropy for mask pixels, and Dice for mask shape quality. A Hungarian matching algorithm assigns predicted queries to ground-truth segments during each forward pass.

ⓘ HuggingFace training mode: When targets are passed to a HuggingFace Mask2Former model, it returns outputs.loss directly. The Hungarian matching and all three loss components are computed internally — you do not need to implement them manually. 
import torch
from torch.optim import AdamW
from torch.optim.lr_scheduler import CosineAnnealingLR
from transformers import (
    Mask2FormerImageProcessor,
    Mask2FormerForUniversalSegmentation,
)


DEVICE = "cuda" if torch.cuda.is_available() else "cpu"
EPOCHS = 40

# ---------------------------------------------------------------
# Load pretrained model for fine-tuning
# ---------------------------------------------------------------
MODEL_NAME = "facebook/mask2former-swin-small-coco-panoptic"
processor  = Mask2FormerImageProcessor.from_pretrained(MODEL_NAME)
model      = Mask2FormerForUniversalSegmentation.from_pretrained(MODEL_NAME)
model      = model.to(DEVICE)

# Fine-tune all parameters; freeze backbone for limited GPU memory.
# Uncomment below to freeze the Swin Transformer backbone:
# for param in model.model.pixel_level_module.encoder.parameters():
#     param.requires_grad = False

optimizer = AdamW(
    [p for p in model.parameters() if p.requires_grad],
    lr=1e-4, weight_decay=1e-2
)
scheduler = CosineAnnealingLR(optimizer, T_max=EPOCHS, eta_min=1e-6)

best_loss = float("inf")

for epoch in range(EPOCHS):
    model.train()
    epoch_loss, n_batches = 0.0, 0

    for batch in train_loader:
        # Processor prepares pixel_values and a formatted mask_labels list.
        # For custom datasets, prepare targets as:
        #   [{"class_labels": tensor, "mask_labels": tensor}, ...] per image
        pixel_values = batch["pixel_values"].to(DEVICE)
        mask_labels  = [m.to(DEVICE) for m in batch["mask_labels"]]
        class_labels = [c.to(DEVICE) for c in batch["class_labels"]]

        # Forward pass — returns outputs.loss (CE + mask BCE + Dice, weighted)
        outputs = model(
            pixel_values=pixel_values,
            mask_labels=mask_labels,
            class_labels=class_labels,
        )

        loss = outputs.loss
        # For debugging, also check auxiliary losses if enabled in the model config.
        # outputs.auxiliary_logits contains intermediate decoder outputs.

        optimizer.zero_grad()
        loss.backward()
        # Clip gradients — transformer models can have large gradient norms early in training.
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=0.01)
        optimizer.step()

        epoch_loss += loss.item()
        n_batches  += 1

    scheduler.step()
    avg_loss = epoch_loss / max(n_batches, 1)
    print(f"Epoch {epoch+1:03d}/{EPOCHS} | loss {avg_loss:.4f} | "
          f"lr {scheduler.get_last_lr()[0]:.2e}")

    if avg_loss < best_loss:
        best_loss = avg_loss
        model.save_pretrained("panoptic_best/")
        processor.save_pretrained("panoptic_best/")
        print(f"  Saved checkpoint at epoch {epoch+1}")
  • Mask2Former's three loss components (CE + mask BCE + Dice) are weighted and summed internally — outputs.loss is the combined scalar to backpropagate.
  • Hungarian matching assigns each predicted query to the closest GT segment; unmatched predictions receive a background class penalty.
  • Use a very small gradient clip norm (0.01 to 0.1) for transformer-based models to avoid training instability.
  • Freeze the backbone encoder when GPU memory is limited — only the transformer decoder and heads need to learn domain-specific features.
  • Use model.save_pretrained() to save HuggingFace models so they reload cleanly with from_pretrained().

Step 4 — Panoptic Quality Evaluation

Panoptic Quality (PQ) evaluates both the quality of matched masks (SQ) and the model's ability to find the right set of segments (RQ). This step implements segment-level IoU matching and the full PQ formula from scratch so you can understand every component.

import torch
import numpy as np
from collections import defaultdict


# ---------------------------------------------------------------
# Helper: segment-level IoU between two boolean masks
# ---------------------------------------------------------------
def segment_iou(pred_mask, gt_mask, eps=1e-7):
    """
    Compute IoU between two boolean tensors of the same shape (H, W).
    Uses element-wise AND for intersection and OR for union.
    """
    intersection = (pred_mask & gt_mask).sum().float()
    union        = (pred_mask | gt_mask).sum().float()
    return (intersection / (union + eps)).item()


# ---------------------------------------------------------------
# Per-category PQ accumulator
# ---------------------------------------------------------------
def compute_pq(predicted_segments, ground_truth_segments, iou_thresh=0.50):
    """
    Args:
        predicted_segments: dict {segment_id: {"mask": BoolTensor(H,W), "category_id": int}}
        ground_truth_segments: same structure
        iou_thresh: IoU threshold for a match to count as TP (COCO standard: 0.5)
    Returns:
        dict with per-category and overall PQ, SQ, RQ
    """
    # Accumulate TP/FP/FN and summed IoU per category
    stats = defaultdict(lambda: {"tp": 0, "fp": 0, "fn": 0, "iou_sum": 0.0})
    used_gt_ids = set()

    # --- Match predictions to ground truth ---
    for pred_id, pred_info in predicted_segments.items():
        cat   = pred_info["category_id"]
        pmask = pred_info["mask"]
        best_iou, best_gt_id = 0.0, None

        for gt_id, gt_info in ground_truth_segments.items():
            if gt_id in used_gt_ids:
                continue
            # Only match within the same category
            if gt_info["category_id"] != cat:
                continue
            iou = segment_iou(pmask, gt_info["mask"])
            if iou > best_iou:
                best_iou, best_gt_id = iou, gt_id

        if best_iou >= iou_thresh:
            # True positive: matched prediction with sufficient overlap
            stats[cat]["tp"]      += 1
            stats[cat]["iou_sum"] += best_iou
            used_gt_ids.add(best_gt_id)
        else:
            # False positive: prediction with no matching GT
            stats[cat]["fp"] += 1

    # --- Count false negatives (GT segments not matched) ---
    for gt_id, gt_info in ground_truth_segments.items():
        if gt_id not in used_gt_ids:
            stats[gt_info["category_id"]]["fn"] += 1

    # --- Compute PQ = SQ x RQ per category ---
    results = {}
    all_pq, all_sq, all_rq = [], [], []

    for cat, s in stats.items():
        tp, fp, fn = s["tp"], s["fp"], s["fn"]

        # SQ: average IoU of matched segments (quality of the masks)
        sq = s["iou_sum"] / (tp + 1e-7) if tp > 0 else 0.0

        # RQ: F1-like recognition score (were the right segments found?)
        rq = tp / (tp + 0.5 * fp + 0.5 * fn + 1e-7)

        # PQ: combined score — a mask must be both well-shaped AND correctly detected
        pq = sq * rq

        results[cat] = {"PQ": pq, "SQ": sq, "RQ": rq,
                        "TP": tp, "FP": fp, "FN": fn}
        all_pq.append(pq); all_sq.append(sq); all_rq.append(rq)

    results["overall"] = {
        "PQ": np.mean(all_pq) if all_pq else 0.0,
        "SQ": np.mean(all_sq) if all_sq else 0.0,
        "RQ": np.mean(all_rq) if all_rq else 0.0,
    }
    return results


# ---------------------------------------------------------------
# Run PQ evaluation on the validation set
# ---------------------------------------------------------------
DEVICE    = "cuda" if torch.cuda.is_available() else "cpu"
MODEL_DIR = "panoptic_best/"

processor = Mask2FormerImageProcessor.from_pretrained(MODEL_DIR)
model     = Mask2FormerForUniversalSegmentation.from_pretrained(MODEL_DIR)
model.eval().to(DEVICE)

accumulated_pq = defaultdict(lambda: {"tp": 0, "fp": 0, "fn": 0, "iou_sum": 0.0})

with torch.no_grad():
    for batch in val_loader:
        images_pil   = batch["images_pil"]    # list of PIL images
        gt_targets   = batch["panoptic_targets"]  # list of GT segment dicts

        inputs   = processor(images=images_pil, return_tensors="pt").to(DEVICE)
        outputs  = model(**inputs)

        panoptic_results = processor.post_process_panoptic_segmentation(
            outputs,
            target_sizes=[img.size[::-1] for img in images_pil],
        )

        for result, gt in zip(panoptic_results, gt_targets):
            seg_map  = result["segmentation"].cpu()
            pred_seg = {}
            for seg in result["segments_info"]:
                pred_seg[seg["id"]] = {
                    "mask":        seg_map == seg["id"],
                    "category_id": seg["label_id"],
                }
            pq_result = compute_pq(pred_seg, gt["segments"])
            # Accumulate per-category stats across all images
            for cat, s in pq_result.items():
                if cat == "overall":
                    continue
                accumulated_pq[cat]["tp"]      += s["TP"]
                accumulated_pq[cat]["fp"]      += s["FP"]
                accumulated_pq[cat]["fn"]      += s["FN"]
                accumulated_pq[cat]["iou_sum"] += s["SQ"] * s["TP"]

# Final PQ summary across all validation images
print("\n=== Panoptic Quality Results ===")
pq_vals = []
for cat, s in accumulated_pq.items():
    tp = s["tp"]; fp = s["fp"]; fn = s["fn"]
    sq = s["iou_sum"] / (tp + 1e-7) if tp > 0 else 0.0
    rq = tp / (tp + 0.5*fp + 0.5*fn + 1e-7)
    pq = sq * rq
    pq_vals.append(pq)
    print(f"  Category {cat:3d}: PQ={pq:.3f}  SQ={sq:.3f}  RQ={rq:.3f}  "
          f"TP={tp}  FP={fp}  FN={fn}")
print(f"\n  Mean PQ (all categories): {np.mean(pq_vals):.4f}")
ⓘ Reading the results: A low SQ with high RQ means the model finds the right segments but predicts poorly shaped masks. A high SQ with low RQ means the masks are precise but many segments are missed or spuriously added. Both components must be high for a good PQ score.
  • PQ = SQ × RQ — a segment must be both well-shaped (high SQ) and correctly detected (high RQ) to contribute to PQ.
  • Segments are matched within the same category only — a predicted car cannot match a GT pedestrian even if their masks overlap.
  • The IoU threshold for a TP match is 0.5 by default in the COCO panoptic benchmark.
  • Accumulate TP/FP/FN across the entire validation set before computing final PQ — per-image averages can produce misleading results when segment counts vary.
