Panoptic Segmentation

Panoptic segmentation unifies semantic and instance segmentation. Every pixel receives a semantic class, and countable objects also receive instance identities.

Panoptic Segmentation Definition

A panoptic output assigns every pixel to exactly one segment. Background-like regions such as road, sky, grass, tissue, or soil are labeled as stuff. Countable objects such as people, cars, fruits, cells, or tools are labeled as things with instance identities.

Pixel assignment P(i,j) = (c, id), where id = 0 for stuff and id > 0 for thing instances Constraint Each pixel belongs to one and only one panoptic segment
Definition: Panoptic segmentation is a unified scene parsing task: it labels stuff regions and separates thing instances in one non-overlapping output.

Things vs Stuff Classes

The thing-stuff distinction is central to panoptic datasets. Things are countable objects with individual identities. Stuff is amorphous material or background context that usually does not need an instance id.

Things

Cars, pedestrians, leaves, fruits, tools, animals, lesions, and cell nuclei. These categories can be counted and separated into individual instances.

Stuff

Road, sky, water, soil, grass, wall, tissue background, and other continuous regions. These categories are labeled as semantic regions.

Intuition: If asking "how many?" makes sense, the category is often a thing. If the region is better described as material, surface, or background, it is often stuff.

Relationship with Semantic and Instance Segmentation

Panoptic segmentation can be understood as a disciplined merge of semantic segmentation and instance segmentation. It keeps the full pixel coverage of semantic segmentation while preserving individual identities for thing classes.

Panoptic Quality

Panoptic Quality evaluates recognition and mask quality together. It matches predicted and ground-truth segments, measures overlap for matched pairs, and penalizes missed or extra segments.

Panoptic Quality PQ = SQ × RQ Segmentation Quality SQ = average IoU over matched segments Recognition Quality RQ = TP / (TP + 0.5 FP + 0.5 FN)
Exam note: PQ is not just IoU. It combines the quality of predicted masks with the model's ability to find the correct set of segments.

Applications

Autonomous Driving

Unify lane, road, sky, person, vehicle, sign, and sidewalk understanding in one scene representation.

Robotics

Support navigation and manipulation by separating objects while retaining layout and background context.

Agriculture

Distinguish crop rows, soil, weeds, fruits, leaves, and individual plants for field analytics.

Medical Imaging

Represent tissue regions and countable anatomical or cellular instances in a single prediction map.

PyTorch Tutorial Lab

These examples show how panoptic labels are prepared, how a universal segmentation model can be loaded, and how Panoptic Quality logic is implemented.

Preparing Panoptic Targets

Libraries: PyTorch, NumPy. A panoptic target stores a semantic class and an instance id for each segment.

import torch

height, width = 512, 512

# Example target tensors for one image.
semantic_map = torch.zeros((height, width), dtype=torch.long)
instance_map = torch.zeros((height, width), dtype=torch.long)

# Segment 1: road as stuff class.
road_pixels = semantic_map == 0
semantic_map[road_pixels] = 7       # class id for road
instance_map[road_pixels] = 0       # stuff has no instance id

# Segment 2: first car as thing class.
car_1_mask = torch.zeros((height, width), dtype=torch.bool)
car_1_mask[220:300, 140:240] = True
semantic_map[car_1_mask] = 2        # class id for car
instance_map[car_1_mask] = 1        # first car instance

target = {
    "semantic_map": semantic_map,
    "instance_map": instance_map
}

Explanation: Panoptic labels must avoid overlapping assignments. Every pixel should have exactly one semantic class, and thing pixels should also have an instance id.

Mask2Former Panoptic Inference

Libraries: Transformers, PyTorch, Pillow. Mask2Former predicts masks and classes through a mask classification formulation.

from PIL import Image
import torch
from transformers import Mask2FormerImageProcessor, Mask2FormerForUniversalSegmentation

image = Image.open("street_scene.jpg").convert("RGB")

processor = Mask2FormerImageProcessor.from_pretrained(
    "facebook/mask2former-swin-small-coco-panoptic"
)
model = Mask2FormerForUniversalSegmentation.from_pretrained(
    "facebook/mask2former-swin-small-coco-panoptic"
)

inputs = processor(images=image, return_tensors="pt")

with torch.no_grad():
    outputs = model(**inputs)

panoptic = processor.post_process_panoptic_segmentation(
    outputs,
    target_sizes=[image.size[::-1]]
)[0]

segmentation_map = panoptic["segmentation"]
segments_info = panoptic["segments_info"]
print(segments_info[:3])

Explanation: Post-processing converts model logits into a panoptic map and metadata for each segment, including class labels and instance ids.

Core Panoptic Quality Calculation

Libraries: PyTorch. Real benchmarks include category matching and ignore labels; this compact version shows the core idea.

import torch

def segment_iou(pred_segment, gt_segment, eps=1e-7):
    intersection = (pred_segment & gt_segment).sum().float()
    union = (pred_segment | gt_segment).sum().float()
    return intersection / (union + eps)

matched_ious = []
tp, fp, fn = 0, 0, 0
used_gt = set()

for pred_id, pred_mask in predicted_segments.items():
    best_gt_id, best_iou = None, 0.0

    for gt_id, gt_mask in ground_truth_segments.items():
        if gt_id in used_gt:
            continue
        iou = segment_iou(pred_mask, gt_mask)
        if iou > best_iou:
            best_gt_id, best_iou = gt_id, iou

    if best_iou > 0.50:
        tp += 1
        used_gt.add(best_gt_id)
        matched_ious.append(best_iou)
    else:
        fp += 1

fn = len(ground_truth_segments) - len(used_gt)
sq = torch.stack(matched_ious).mean() if matched_ious else torch.tensor(0.0)
rq = tp / (tp + 0.5 * fp + 0.5 * fn + 1e-7)
pq = sq * rq

Explanation: PQ rewards good masks and correct recognition. False positives and false negatives reduce RQ even when matched masks have high IoU.

Panoptic Training Skeleton

Libraries: PyTorch. Universal segmentation models often combine classification, mask, and matching losses.

import torch

model.train()

for batch in train_loader:
    images = batch["images"].to(device)
    targets = batch["targets"]

    # The model returns losses for masks, classes, and matching.
    outputs = model(images, targets=targets)
    loss_dict = outputs.losses

    total_loss = (
        loss_dict["loss_ce"]
        + loss_dict["loss_mask"]
        + loss_dict["loss_dice"]
    )

    optimizer.zero_grad()
    total_loss.backward()
    optimizer.step()

    print({
        "class": float(loss_dict["loss_ce"]),
        "mask": float(loss_dict["loss_mask"]),
        "dice": float(loss_dict["loss_dice"])
    })

Explanation: A panoptic model must learn what each mask represents and where the mask is located. Matching losses help assign predicted masks to ground-truth segments during training.
