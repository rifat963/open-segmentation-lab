Instance Segmentation

Instance segmentation detects individual objects and predicts a separate mask for each object. It is useful when counting, tracking, measuring, or manipulating individual objects matters.

Instance Segmentation Definition

Instance segmentation combines object localization, class prediction, and pixel-level mask estimation. Instead of assigning one class label to every pixel, the model predicts a set of object instances. Each instance usually contains a class label, confidence score, bounding box, and binary mask.

Output set S = {(ck, sk, bk, Mk)}k=1...N Mask shape Mk ∈ {0,1}H × W
Definition: If an image contains five concrete cracks, five leaves, or five cells from the same class, instance segmentation should produce five separate masks rather than one merged foreground region.

Detection, Semantic Segmentation, and Instance Segmentation

Object detection predicts rectangular boxes, while instance segmentation predicts object shapes. Semantic segmentation is dense and class-based, but it merges neighboring objects of the same class. Instance segmentation is object-based and identity-aware.

Task Output Best question Typical metric
Object detectionBoxes and labelsWhere is the object?Box AP
Semantic segmentationClass mapWhich pixels belong to each class?mIoU, Dice
Instance segmentationObject masksWhich pixels belong to each object?Mask AP, Mask IoU
Intuition: Semantic segmentation says "these pixels are cars." Instance segmentation says "these pixels are car 1, those pixels are car 2."

Important Model Families

Mask R-CNN

Extends Faster R-CNN with a mask branch after RoIAlign. It is a strong academic baseline because its proposal, classification, box regression, and mask stages are easy to inspect.

YOLACT

Separates mask generation into prototype masks and instance-specific coefficients, making real-time instance segmentation more practical.

YOLO Segmentation

Combines fast object detection with mask prediction, often used when speed and simple deployment matter in applied computer vision projects.

SAM-assisted Annotation

Uses promptable masks from points or boxes to accelerate labeling. Students should audit these masks because promptable masks are not automatically class-specific ground truth.

Mask IoU and Mask AP

Mask IoU measures overlap between one predicted instance mask and one ground-truth instance mask. Mask AP evaluates a ranked set of predictions across confidence and IoU thresholds.

Mask IoU IoU(M, G) = |M ∩ G| / |M ∪ G| Detection quality AP = area under the precision-recall curve over matched masks
Pitfall: A model can detect the correct object with a good box but still predict a poor mask. Always inspect visual masks and mask metrics, not only detection scores.

Mask R-CNN Fine-tuning with Torchvision

Libraries: PyTorch, Torchvision. The dataset must return an image and a target dictionary containing boxes, labels, and masks.

import torch
from torch.utils.data import DataLoader
from torchvision.models.detection import maskrcnn_resnet50_fpn
from torchvision.models.detection.faster_rcnn import FastRCNNPredictor
from torchvision.models.detection.mask_rcnn import MaskRCNNPredictor

num_classes = 3  # background + class 1 + class 2

# Load a COCO-pretrained Mask R-CNN backbone.
model = maskrcnn_resnet50_fpn(weights="DEFAULT")

# Replace the box classifier for the target dataset.
in_features = model.roi_heads.box_predictor.cls_score.in_features
model.roi_heads.box_predictor = FastRCNNPredictor(in_features, num_classes)

# Replace the mask predictor for the target dataset.
in_channels = model.roi_heads.mask_predictor.conv5_mask.in_channels
hidden_layer = 256
model.roi_heads.mask_predictor = MaskRCNNPredictor(
    in_channels,
    hidden_layer,
    num_classes
)

device = "cuda" if torch.cuda.is_available() else "cpu"
model.to(device)
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4)

for images, targets in DataLoader(train_dataset, batch_size=2, collate_fn=lambda x: tuple(zip(*x))):
    images = [img.to(device) for img in images]
    targets = [{key: value.to(device) for key, value in t.items()} for t in targets]

    # During training, Mask R-CNN returns a dictionary of losses.
    loss_dict = model(images, targets)
    loss = sum(loss_value for loss_value in loss_dict.values())

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

Explanation: The model learns proposal quality, class labels, bounding boxes, and masks together. For small datasets, start from COCO weights and replace only the prediction heads.

YOLO-style Segmentation Training

Libraries: Ultralytics YOLO. Labels are commonly stored as polygon masks in a YOLO segmentation dataset format.

from ultralytics import YOLO

# Start from a pretrained segmentation checkpoint.
model = YOLO("yolov8n-seg.pt")

# data.yaml should define train/val image folders and class names.
model.train(
    data="data.yaml",
    epochs=80,
    imgsz=640,
    batch=8,
    lr0=0.001,
    patience=20
)

# Run inference and save visual mask overlays.
results = model.predict(
    source="test_images/",
    conf=0.35,
    save=True,
    save_txt=True
)

for result in results:
    # result.masks contains predicted instance masks.
    print(result.boxes.cls, result.boxes.conf)

Explanation: YOLO-style segmentation is useful for real-time applied systems. Use it when fast inference is a priority, then compare accuracy with a stronger but slower model such as Mask R-CNN.

SAM-assisted Pseudo Mask Generation

Libraries: Segment Anything, OpenCV, NumPy. Use generated masks as candidates, then filter and manually verify them before training.

import cv2
from segment_anything import SamAutomaticMaskGenerator, sam_model_registry

image = cv2.imread("image.jpg")
image_rgb = cv2.cvtColor(image, cv2.COLOR_BGR2RGB)

sam = sam_model_registry["vit_b"](checkpoint="sam_vit_b.pth")
mask_generator = SamAutomaticMaskGenerator(
    model=sam,
    points_per_side=32,
    pred_iou_thresh=0.88,
    stability_score_thresh=0.92
)

masks = mask_generator.generate(image_rgb)

filtered_masks = []
for mask_info in masks:
    area = mask_info["area"]
    score = mask_info["predicted_iou"]

    # Remove very small or low-confidence candidate masks.
    if area > 300 and score > 0.90:
        filtered_masks.append(mask_info["segmentation"])

print(f"Kept {len(filtered_masks)} candidate instance masks")

Explanation: SAM can reduce annotation time, but generated masks are class-agnostic. A human or a classifier must still assign class semantics and remove incorrect masks.

Mask IoU Matching for Evaluation

Libraries: PyTorch. This function computes pairwise IoU between predicted and ground-truth masks before matching instances.

import torch

def pairwise_mask_iou(pred_masks, gt_masks, eps=1e-7):
    # pred_masks: [N, H, W], gt_masks: [M, H, W]
    pred = pred_masks.bool().flatten(1)
    gt = gt_masks.bool().flatten(1)

    intersection = (pred[:, None, :] & gt[None, :, :]).sum(dim=-1).float()
    union = (pred[:, None, :] | gt[None, :, :]).sum(dim=-1).float()
    return intersection / (union + eps)

iou_matrix = pairwise_mask_iou(pred_masks, gt_masks)
matched_pred, matched_gt = torch.where(iou_matrix >= 0.50)

true_positives = len(matched_pred)
false_positives = pred_masks.shape[0] - true_positives
false_negatives = gt_masks.shape[0] - true_positives

Explanation: Mask AP uses this matching idea repeatedly across IoU thresholds and confidence thresholds, then summarizes the precision-recall behavior.
