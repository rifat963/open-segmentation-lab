Segmentation Metrics

Segmentation metrics compare predicted masks with ground-truth masks. A good evaluation report should include the metric definition, the task it supports, a worked example, and its known limitation. Use the interactive calculators in the sidebar to build intuition.

IoU — Intersection over Union

Intersection over Union measures how much a predicted mask overlaps with the ground-truth mask relative to their combined area. It penalizes both false positives (predicted but wrong) and false negatives (missed pixels).

Intersection over Union IoU = |P ∩ G| / |P ∪ G| = TP / (TP + FP + FN)

Worked Example

A predicted mask covers 80 pixels. The ground-truth mask covers 100 pixels. They share an overlap of 60 pixels.

Union = 80 + 100 − 60 = 120 pixels

IoU = 60 / 120 = 0.50 — the model correctly captured half the true region relative to everything predicted or true.

When to use IoU: Use IoU for semantic and binary segmentation benchmarks. It is the standard per-class metric and the threshold used for matching in Mask AP. An IoU of 0.50 is a common "acceptable match" threshold; 0.75 is strict.

Dice Score (F1 Score)

Dice score doubles the intersection and divides by the sum of predicted and ground-truth areas. It is the harmonic mean of precision and recall at the pixel level. Dice is mathematically related to IoU: Dice = 2·IoU / (1 + IoU).

Dice / F1 Score Dice = 2|P ∩ G| / (|P| + |G|) = 2·TP / (2·TP + FP + FN)

Worked Example (same scenario as IoU)

Predicted = 80 pixels, Ground-truth = 100 pixels, Overlap = 60 pixels.

Dice = 2 × 60 / (80 + 100) = 120 / 180 ≈ 0.667

Dice is higher than IoU (0.50) because it double-weights the overlap. The two metrics rank models identically but report different absolute values.

When to use Dice: Prefer Dice when foreground regions are small relative to the image (medical segmentation, crack detection). Dice is less sensitive to the large correct-background area that inflates pixel accuracy.

Pixel Accuracy

Pixel accuracy measures the fraction of correctly classified pixels across the entire image. It is the simplest metric and the easiest to explain, but it is often misleading when the background or a dominant class fills most of the image.

Pixel Accuracy PA = (Σc TPc) / (H × W)

Worked Example

A 100×100 image has 9 800 background pixels and 200 foreground pixels. A model that predicts everything as background would achieve 98% pixel accuracy while missing the entire object of interest.

This is why pixel accuracy alone is insufficient for imbalanced segmentation tasks.

When to use Pixel Accuracy: Acceptable as a supplementary metric when classes are roughly balanced. Always pair it with mIoU or Dice. Never use it as the sole metric for medical, agricultural, or crack detection tasks where the foreground is small.

Mean IoU (mIoU)

Mean IoU computes IoU for each semantic class separately, then averages across all classes. This prevents large background or dominant classes from masking poor performance on smaller categories.

Mean IoU mIoU = (1/C) Σc=1C IoUc

Worked Example

A 3-class semantic segmentation model reports: Road IoU = 0.92, Car IoU = 0.71, Pedestrian IoU = 0.48.

mIoU = (0.92 + 0.71 + 0.48) / 3 = 0.703

Pedestrians are rare but count equally. A model that ignores pedestrians entirely would score 0 for that class, dragging mIoU down even if road and car performance is excellent.

When to use mIoU: The standard primary metric for multi-class semantic segmentation benchmarks (Cityscapes, ADE20K, PASCAL VOC). Always report mIoU in segmentation papers alongside per-class IoU for rare categories.

Mask AP — Mask Average Precision

Mask Average Precision evaluates instance segmentation by ranking predicted masks by confidence score and checking whether each prediction matches a ground-truth instance at multiple IoU thresholds (0.50 to 0.95 in COCO evaluation). It combines detection quality and mask quality into a single number.

Mask Average Precision (COCO standard) AP = (1/T) Σt APIoU=t,  t ∈ {0.50, 0.55, …, 0.95}

Key thresholds

AP@0.50 (AP50): A predicted mask must overlap the ground-truth by at least 50% to count as a true positive. Lenient.

AP@0.75 (AP75): Requires 75% overlap. Strict; rewards well-fitted masks.

AP (COCO): Average of AP@0.50 to AP@0.95 in steps of 0.05. The standard COCO benchmark number.

Pitfall: Mask AP depends on confidence ranking as well as mask overlap. A model with good masks but poorly calibrated confidence scores will lose AP points. Always inspect visual masks and do not rely on AP alone to understand mask quality.
When to use Mask AP: The standard metric for instance segmentation benchmarks (COCO). Use it whenever objects must be individually detected and masked. Do not use it for semantic segmentation, where there is no instance identity.

Panoptic Quality (PQ)

Panoptic Quality combines segmentation quality and recognition quality into a single metric. It is the standard evaluation for panoptic segmentation where every pixel must be assigned both a class label and, for countable objects, an instance identity.

Panoptic Quality PQ = SQ × RQ Segmentation Quality SQ = (1/|TP|) Σ(p,g) ∈ TP IoU(p, g) Recognition Quality RQ = TP / (TP + ½ FP + ½ FN)

How PQ is computed

Step 1 — Match: A predicted segment is matched to a ground-truth segment if and only if their IoU exceeds 0.50. Only one prediction can match each ground truth.

Step 2 — SQ (Segmentation Quality): Average IoU of all matched pairs. Rewards mask precision.

Step 3 — RQ (Recognition Quality): F1 score over matched vs. unmatched segments. Rewards coverage.

PQ = SQ × RQ. A model that matches well but misses instances scores high SQ but low RQ, giving a low PQ.

When to use PQ: Required for panoptic segmentation benchmarks (COCO-Panoptic, Cityscapes Panoptic). Also reported per stuff and per things separately (PQSt and PQTh) to distinguish semantic and instance quality.

Metric Comparison Summary

Each metric suits a specific task and evaluation goal. Choosing the right metric is as important as choosing the right model.

Metric Primary Task Key Strength Key Limitation
IoU Semantic & instance (per mask) Penalizes both FP and FN; the standard overlap ratio Harsh on tiny objects; sensitive to boundary uncertainty
Dice Score Medical & binary segmentation Robust for small foreground regions; weighted overlap Less intuitive for multi-instance evaluation
Pixel Accuracy Semantic segmentation Simple, easy to explain, cheap to compute Inflated by large background; misleading for imbalanced data
mIoU Multiclass semantic segmentation Balances class-wise performance; standard benchmark metric Rare classes with few pixels produce unstable per-class estimates
Mask AP Instance segmentation Evaluates detection ranking and mask quality together Complex to compute; affected by confidence calibration
Panoptic Quality Panoptic segmentation Unifies recognition and overlap quality in one number Requires careful segment matching and panoptic ground-truth labels
